# Система анализа результатов нагрузочного тестирования

Веб-приложение для анализа результатов нагрузочного тестирования.

## Что умеет приложение

- управление проектами, профилями НТ, тестами и отчетами;
- ролевая модель:
  - `ADMIN`
  - `ENGINEER`
  - `CUSTOMER`
- формирование `целевого` отчета по одному тесту;
- формирование `сравнительного` отчета по двум тестам;
- анализ:
  - превышений SLA;
  - ошибок выполнения;
  - попадания в профиль НТ;
  - утилизации CPU / RAM / DISK;
- добавление графиков из `Grafana` в HTML-отчет;
- сохранение отчетов и последующий просмотр в интерфейсе;
- дашборд на главной странице с актуальной статистикой по проекту.

## Архитектура

Приложение использует:

- `FastAPI` — backend и маршруты;
- `Jinja2 + Bootstrap` — HTML-интерфейс;
- `PostgreSQL` — основная локальная база данных приложения;
- `InfluxDB` — иточник бизнес-метрик нагрузочного тестирования;
- `Prometheus` — источник аппаратных метрик;
- `Grafana` — визуализация и рендер графиков в отчеты.

## Роли пользователей

### ADMIN

- видит все проекты, тесты, профили, отчеты и пользователей;
- управляет пользователями и шаблонами отчетов;
- может создавать, редактировать и удалять тесты и отчеты.

### ENGINEER

- работает только в рамках своего проекта;
- может создавать тесты и формировать отчеты;
- видит статистику и историю только своего проекта.

### CUSTOMER

- работает только в рамках своего проекта;
- может просматривать тесты и отчеты;
- не может формировать или удалять отчеты.

## Источники данных

### PostgreSQL

Используется как локальная БД на хост-машине.

Приложение внутри Docker подключается к ней через:

- `POSTGRES_HOST=host.docker.internal`

### InfluxDB

Используется для бизнес-метрик


### Prometheus

Используется для аппаратных метрик

### Grafana

Используется для:

- просмотра дашбордов;
- рендера графиков в отчеты;
- перехода из отчета к полному дашборду по окну теста.

## Быстрый старт

### 1. Подготовить `.env`

Минимально важные параметры:

```env
POSTGRES_DB=НАЗВАНИЕ_БД
POSTGRES_USER=ПОЛЬЗОВАТЕЛЬ_POSTGRES
POSTGRES_PASSWORD=ПАРОЛЬ_POSTGRES
POSTGRES_HOST=ХОСТ_POSTGRES
POSTGRES_PORT=5432

```

### 2. Запустить сервисы

```bash
docker compose up --build
```

### 3. Открыть сервисы

- приложение: `http://localhost:8000`
- Grafana: `http://localhost:3000`
- InfluxDB: `http://localhost:8086`
- Prometheus: `http://localhost:9090`

## Учетные данные по умолчанию

После первого запуска создается администратор:

- логин: `admin`
- пароль: `admin123`

Значения можно изменить через `.env`:

- `DEFAULT_ADMIN_LOGIN`
- `DEFAULT_ADMIN_PASSWORD`

## Docker-сервисы

В `docker-compose.yml` поднимаются:

- `app`
- `grafana`
- `renderer`
- `influxdb`
- `prometheus`


## Отчеты

### Целевой отчет

Формируется по одному тесту и включает:

- статус теста;
- основные выводы;
- сводку по времени отклика;
- попадание в профиль;
- общую сводку по операциям;
- бизнес-графики;
- аппаратные метрики;
- ссылку на Grafana;
- подпись пользователя, сформировавшего отчет.

### Сравнительный отчет

Формируется по двум тестам и включает:

- вывод о деградации / улучшении / отсутствии изменений;
- краткую сводку по ключевым критериям;
- сравнение времени отклика;
- сравнение попадания в профиль;
- сравнение ошибок;
- сравнение аппаратных метрик;
- ссылки на Grafana;
- подпись пользователя, сформировавшего отчет.

## Синтетические данные

- генераторы данных;
